## 1. Agent 定义与核心能力

### 1.1 什么是 AI Agent？

**定义**：AI Agent 是具备**环境感知、自主决策、目标驱动、工具执行、记忆迭代、反思优化**全闭环能力的智能实体，能够在无人工持续干预的情况下，自主完成多步骤、高复杂度的开放域任务。

### 1.2 Agent vs LLM vs RAG

| 维度 | 传统 LLM | RAG | AI Agent |
|------|----------|-----|-----------|
| 核心定位 | 推理大脑 | 记忆增强工具 | 完整智能体 |
| 自主决策 | ❌ | ❌ | ✅ |
| 工具执行 | ❌ | ❌ | ✅ |
| 记忆能力 | 有限上下文 | 外部知识库 | 多层记忆系统 |
| 反思迭代 | ❌ | ❌ | ✅ |
| 任务执行 | 被动问答 | 检索增强 | 主动规划 |

### 1.3 Agent 的核心工作流

```
┌─────────────────────────────────────────────────────────────────┐
│                    Agent R-A-O 循环                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│     ┌──────────┐                                                 │
│     │  OBSERVE │ ←── 获取环境信息（用户输入、工具返回）           │
│     └────┬─────┘                                                 │
│          ↓                                                       │
│     ┌──────────┐                                                 │
│     │ REASON   │ ←── 基于目标和上下文推理下一步行动               │
│     └────┬─────┘                                                 │
│          ↓                                                       │
│     ┌──────────┐                                                 │
│     │   ACT    │ ←── 执行行动（调用工具、生成响应）               │
│     └────┬─────┘                                                 │
│          ↓                                                       │
│     ┌──────────┐                                                 │
│     │  LOOP?   │ ←── 判断任务是否完成，否则继续循环               │
│     └──────────┘                                                 │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## 3. 推理模式：CoT / ReAct / ToT

### 3.1 Chain-of-Thought (CoT)

**核心思想**：让模型显式生成推理步骤，再给出最终答案。

**数学表示**：
$$
P(\text{answer} \mid \text{problem}) = \sum_{\text{chain} \in \mathcal{C}} P(\text{chain} \mid \text{problem}) \cdot P(\text{answer} \mid \text{chain})
$$

**示例**：
```
问题：小明有 5 个苹果，小红给了他 3 个，小明吃掉了 2 个，还剩多少？
推理链：
1. 小明开始有 5 个苹果
2. 小红给了 3 个 → 5 + 3 = 8 个
3. 小明吃掉 2 个 → 8 - 2 = 6 个
答案：6 个
```

### 3.2 ReAct (Reason + Act)

**核心思想**：在推理过程中**交替进行思考和行动**。

ReAct Loop：
```
Thought: 我需要...
Action: tool_name[input]
Observation: 工具返回了...
Thought: 基于观察，...
Action: ...
...
```

**与 CoT 的区别**：
- CoT：纯推理，不与外部交互
- ReAct：推理 + 行动，可获取外部信息

### 3.3 Tree of Thoughts (ToT)

**核心思想**：将推理扩展为树搜索，支持**回溯和多路径探索**。

```
        问题
       /    \
    路径A   路径B
    / \      / \
   A1  A2   B1  B2
```

**适用场景**：需要探索和规划的复杂任务。

### 3.4 对比

| 模式 | 搜索策略 | 行动能力 | 适用场景 |
|------|----------|---------|---------|
| CoT | 单链 | ❌ | 简单推理 |
| ReAct | 单链 + 工具 | ✅ | 工具使用 |
| ToT | 树搜索 | ✅ | 复杂规划 |

## 4. 记忆机制

### 4.1 记忆类型

| 类型 | 存储内容 | 持续时间 | 容量 |
|------|----------|---------|------|
| **感官记忆** | 原始感知输入 | 毫秒级 | 极低 |
| **工作记忆** | 当前任务相关信息 | 秒-分钟 | 有限 |
| **情景记忆** | 长期经历的事件 | 长期 | 中等 |
| **语义记忆** | 事实和概念 | 长期 | 巨大 |

### 4.2 Agent 中的记忆设计

```
┌─────────────────────────────────────────────────────────────┐
│                    Agent 记忆系统                            │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  ┌─────────────────────────────────────────────────────┐    │
│  │                   长期记忆 (Long-term Memory)          │    │
│  │  ┌────────────────┐  ┌────────────────┐              │    │
│  │  │  语义记忆       │  │  情景记忆       │              │    │
│  │  │  (Semantic)     │  │  (Episodic)    │              │    │
│  │  │  - 知识图谱     │  │  - 历史对话     │              │    │
│  │  │  - 向量数据库   │  │  - 任务轨迹     │              │    │
│  │  └────────────────┘  └────────────────┘              │    │
│  └──────────────────────────┬───────────────────────────┘    │
│                              ↓                               │
│  ┌─────────────────────────────────────────────────────┐    │
│  │                   工作记忆 (Working Memory)           │    │
│  │  - 当前任务上下文                                     │    │
│  │  - 中间推理结果                                       │    │
│  │  - 当前对话历史                                       │    │
│  └──────────────────────────┬───────────────────────────┘    │
│                              ↓                               │
│  ┌─────────────────────────────────────────────────────┐    │
│  │                   反馈记忆 (Feedback Memory)           │    │
│  │  - 执行结果                                           │    │
│  │  - 错误记录                                           │    │
│  └─────────────────────────────────────────────────────┘    │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 4.3 记忆检索策略

**向量检索 + 关键词检索 (混合检索)**：

**RRF (Reciprocal Rank Fusion)**：
$$
\text{RRF}(d) = \sum_{i=1}^{K} \frac{1}{k + \text{rank}_i(d)}
$$

其中 $k$ 通常取 60， $\text{rank}_i(d)$ 是第 $i$ 种检索方法对文档 $d$ 的排名。

## 5. 工具调用 (Tool Use)

### 5.1 Function Calling / Tool Use

让 LLM 调用外部工具扩展能力。

### 5.2 工具 Schema 设计

```json
{
  "name": "search_web",
  "description": "Search the web for information",
  "parameters": {
    "type": "object",
    "properties": {
      "query": {
        "type": "string",
        "description": "The search query"
      },
      "max_results": {
        "type": "integer",
        "description": "Maximum number of results",
        "default": 5
      }
    },
    "required": ["query"]
  }
}
```

### 5.3 常用工具类型

| 工具类型 | 功能 | 示例 |
|----------|------|------|
| **搜索** | 获取实时信息 | Web Search, Wikipedia |
| **代码执行** | 运行代码 | Python, SQL |
| **文件操作** | 读写文件 | Read, Write |
| **API 调用** | 访问外部服务 | Weather, Calendar |
| **数据库** | 数据查询 | SQL, Vector DB |
| **知识库** | 领域知识检索 | RAG |

### 5.4 Tool Use 流程

```
1. User Query → "北京今天天气怎么样？"
          ↓
2. LLM 分析是否需要工具 → 需要，天气查询
          ↓
3. LLM 生成调用请求 → search_weather[city="北京"]
          ↓
4. 工具执行 → 返回 {temp: 25, condition: "晴"}
          ↓
5. LLM 整合结果 → "北京今天气温25℃，天气晴朗"
```

## 7. 高频面试题

### Q1: 什么是 AI Agent？ 与 LLM 的区别？

**参考答案**：

AI Agent 是具备**自主决策、任务拆解、工具调用、反思迭代**能力的智能系统，而 LLM 主要是被动响应输入的文本生成模型。

**核心区别**：
- LLM：被动问答，无自主规划
- Agent：主动规划、执行、迭代

给 LLM 装上"感知系统、记忆系统、行动系统、反思系统"，它就变成了 Agent。

---

### Q2: Agent 的核心工作流是什么？

**参考答案**：

Agent 的核心是 **R-A-O 循环** (Reason-Act-Observe)：

1. **Reason (推理)**：基于任务目标和上下文，拆解任务、决策下一步行动
2. **Act (行动)**：执行推理后的步骤，可调用工具、生成内容
3. **Observe (观察)**：获取行动结果，判断是否符合目标

三者循环往复，直到完成任务。

---

### Q3: ReAct 和 CoT 的区别？

**参考答案**：

| 模式 | 特点 | 与环境交互 |
|------|------|-----------|
| CoT | 生成推理链，不调用工具 | ❌ |
| ReAct | Thought + Action + Observation 交替 | ✅ |

CoT 适合纯推理任务（数学证明），ReAct 适合需要外部信息或工具的任务（搜索、API 调用）。

---

### Q4: Agent 的记忆有哪些类型？

**参考答案**：

| 类型 | 作用 | 实现方式 |
|------|------|---------|
| **工作记忆** | 当前任务上下文 | 有限上下文窗口 |
| **情景记忆** | 历史交互 | 向量数据库 |
| **语义记忆** | 知识概念 | 知识图谱 |
| **程序记忆** | Agent 行为模式 | System Prompt |

### Q5: 如何设计一个 Agent 系统？

**参考答案**：

**步骤 1：需求分析**
- 输入输出定义
- 任务复杂度评估

**步骤 2：组件设计**
- 规划模块（任务拆解）
- 推理模块（ReAct/CoT）
- 记忆模块（向量存储）
- 工具模块（Function Schema）

**步骤 3：执行流程**
```
Thought: 需要...
Action: tool_name[input]
Observation: ...
Thought: ...
...
```

**步骤 4：评估与迭代**
- 准确率、效率、稳定性

---

### Q6: 什么是 Function Calling？

**参考答案**：

Function Calling 是让 LLM **生成结构化的工具调用请求**的机制：

1. 定义工具的 JSON Schema
2. LLM 分析用户意图，选择合适工具
3. 生成 `{name: "tool_name", arguments: {...}}`
4. 执行工具，返回结果
5. LLM 整合结果生成最终响应

---

### Q7: Multi-Agent 如何协作？

**参考答案**：

常见协作模式：

1. **串行模式**：Manager → Worker1 → Worker2 → ... → User
2. **并行模式**：Manager 分解任务，并行分发给多个 Worker
3. **层次模式**：Manager 协调多个专业 Agent

关键设计：
- 角色定义（System Prompt）
- 通信协议（Agent 间消息格式）
- 决策聚合（如何汇总多 Agent 结果）

### Q8: Agent 的评估指标有哪些？

**参考答案**：

| 维度 | 指标 | 说明 |
|------|------|------|
| **准确性** | Task Success Rate | 任务完成度 |
| **效率** | Steps / Time | 执行步骤和时间 |
| **稳定性** | Consistency | 多次执行一致性 |
| **安全性** | Harmful Output Rate | 有害输出比例 |

## 📝 面试高频题 & 参考答案

### Q1: LLM Agent 和传统 Agent 有何本质区别？

**参考答案：**

| 维度 | 传统 Agent（RL-based） | LLM Agent |
|------|----------------------|-----------|
| **决策机制** | 强化学习策略网络 | LLM 自回归生成 + Prompt |
| **知识来源** | 环境交互数据 | 预训练知识 + 工具调用 |
| **泛化能力** | 特定任务 | 开放域任务 |
| **规则注入** | 人工设计 reward | Prompt 引导 |
| **典型代表** | AlphaGo、游戏AI | AutoGPT、ChatGPT Plugins |

**核心区别**：LLM Agent 基于大模型的涌现能力（In-Context Learning、CoT），无需为每个任务单独训练，可通过自然语言指令和工具调用完成复杂任务。

---

### Q2: 什么是 ReAct（Reasoning + Acting）模式？

**参考答案：**

ReAct 是让 LLM 交替进行**推理**和**行动**的 Agent 范式：

```
用户问题
    ↓
Thought: 分析当前状态，决定下一步行动
    ↓
Action: 调用工具（如搜索、计算）
    ↓
Observation: 观察工具返回结果
    ↓
Thought: 基于观察继续推理
    ↓
...（循环直到得到答案）
```

**核心思想**：将 CoT（推理）和 Tool Use（行动）结合，推理帮助决定调用什么工具，工具结果反馈给推理形成闭环。

---

### Q3: Agent 的 Context Window 为何是核心约束？

**参考答案：**

Context Window 限制包括：
1. **对话历史**：多轮对话太长时，早期关键信息可能被遗忘
2. **工具定义**：所有工具的 JSON Schema 都塞在上下文中，工具多了就撑满窗口
3. **Tool Return 结果**：工具返回的内容会占用上下文空间

**工程解决方案**：
- History Compression / Summarization
- 按需加载工具（只加载当前任务相关的工具）
- 分阶段处理（先决策再调用工具）

---

### Q4: Tool Calling 的完整链路是什么？

**参考答案：**

```
1. 工具定义（Tool Definition）
   - 编写 JSON Schema 描述工具名、参数、描述
   - 注入到 System Prompt 中

2. LLM 判断调用（Tool Selection）
   - LLM 分析用户意图，判断是否需要工具
   - 从上下文中选择最合适的工具，填充参数

3. 工具执行（Tool Execution）
   - 在外部环境执行工具代码/API调用
   - 返回 JSON 格式的执行结果

4. 结果整合（Result Integration）
   - 将工具返回内容加入对话上下文
   - LLM 基于完整上下文生成最终回复
```

---

### Q5: System Prompt 在 Agent 系统中的职责是什么？

**参考答案：**

1. **角色定义**：告诉 LLM 扮演什么角色（如"专业分析师"）
2. **行为约束**：定义什么能做、什么不能做
3. **工具描述**：承载所有可用工具的 Schema
4. **输出格式**：规定回复的结构（JSON / Markdown）
5. **对话策略**：如何处理多轮、追问、模糊意图

**如果 System Prompt 越来越长**：
- 按模块拆分，用元 Prompt 动态路由
- 只加载与当前任务相关的工具子集
- 定期将 System Prompt 的静态部分压缩

---

### Q6: 当对话历史太长时，有哪些处理方法？

**参考答案：**

| 方法 | 原理 | 优缺点 |
|------|------|--------|
| **截断（Truncation）** | 保留最近 N 轮 | 简单，但早期信息丢失 |
| **摘要压缩（Summarization）** | 用 LLM 将历史摘要 | 保记忆，但计算开销 |
| **滑动窗口 + 摘要** | 窗口滑动，定期摘要 | 平衡新旧信息 |
| **语义压缩（Semantic Compression）** | 提取关键实体/关系存储 | 保留关键信息，丢失细节 |
| **RAG 化** | 历史也做向量检索 | 适合需要精准召回 |

### Q7: Agent 调用工具返回超大量结果时如何处理？

**参考答案：**

例如代码搜索返回 50KB 结果，撑满上下文。

**解决思路**：
1. **截断**：只取前 N 个最相关结果
2. **流式处理**：工具支持分页，Agent 分批获取
3. **压缩**：在 Tool Return 后加一步"摘要"，压缩后再使用
4. **工具设计**：工具本身应该限制返回量（top_k），不要返回全部

### Q8: 什么是 Agent 的规划（Planning）能力？

**参考答案：**

LLM Agent 的规划指：**将复杂任务分解为多个子步骤，并按合理顺序执行**。

**代表方法**：
- **CoT（Chain-of-Thought）**：逐步推理，每步专注于一个小目标
- **ToT（Tree-of-Thought）**：探索多条分解路径，选择最优
- **LLM Compiler**：动态规划子任务并行执行
- **HuggingGPT**：用 LLM 协调多个专用模型

**面试追问**：ReAct 和 CoT 有何区别？
> CoT 只做推理不调用工具；ReAct 是推理 + 行动交替进行，工具调用是核心环节。
