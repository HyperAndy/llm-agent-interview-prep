## 1. RAG 概述与原理

### 1.1 什么是 RAG？

**RAG (Retrieval-Augmented Generation)**：检索增强生成，将外部知识检索与生成式模型结合的系统。

### 1.2 为什么需要 RAG？

| 问题 | LLM 局限 | RAG 解决方案 |
|------|----------|-------------|
| 知识过时 | 训练数据有截止日期 | 实时检索外部知识 |
| 幻觉 | 可能生成错误内容 | 基于真实文档生成 |
| 长尾知识 | 不常见知识覆盖少 | 精确检索补充 |
| 可溯源性 | 不知道答案来源 | 提供引用来源 |

---

## 2. RAG 架构与流程

### 2.1 完整 RAG 流程

```
用户查询 → 检索阶段 → 生成阶段 → 答案(附引用)
```

### 2.2 各阶段详解

| 阶段 | 输入 | 输出 | 关键技术 |
|------|------|------|----------|
| 文档加载 | PDF/HTML/Markdown | 原始文本 | PyMuPDF, Unstructured |
| 文本切分 | 长文本 | Chunks | 固定大小/语义切分 |
| 向量化 | Chunk | Vector | Embedding Model |
| 向量存储 | Vector | Index | FAISS, Milvus, Pinecone |
| Query改写 | 用户Query | 检索Query | HyDE, Query Expansion |
| 检索 | Query Embedding | Top-K Chunks | ANN, BM25 |
| 重排 | Top-K Chunks | 重排后列表 | Cross-Encoder |
| 生成 | Query + Context | 答案 | LLM |

## 3. 分块策略 (Chunking)

### 3.1 为什么分块重要？

分块直接影响检索质量和生成效果：
- **块太大**：引入噪声，上下文溢出
- **块太小**：丢失语义完整性，检索召回低

### 3.2 分块策略对比

| 策略 | 描述 | 优点 | 缺点 |
|------|------|------|------|
| **固定大小** | 按 token/字符数切分 | 简单高效 | 语义可能断裂 |
| **语义切分** | 按句子/段落切分 | 保持语义 | 实现复杂 |
| **递归切分** | 多级粒度切分 | 灵活 | 调参复杂 |
| **文档结构** | 按标题/章节切分 | 符合写作逻辑 | 依赖文档格式 |

### 3.3 分块大小选择

| 应用场景 | 建议块大小 | 理由 |
|----------|-----------|------|
| 摘要 | 2,000-4,000 | 需要足够上下文 |
| 问答 | 500-1,000 | 精确匹配 |
| 搜索 | 256-512 | 快速检索 |
| 代码 | 128-256 | 函数级完整 |

## 4. 检索技术

### 4.1 稀疏检索 vs 密集检索

| 类型 | 方法 | 优点 | 缺点 |
|------|------|------|------|
| **Sparse (BM25)** | 关键词匹配 + TF-IDF | 可解释、精确匹配 | 无法语义理解 |
| **Dense (Vector)** | 神经网络嵌入 | 语义相似 | 需要足够数据 |

### 4.2 BM25 算法

**Okapi BM25** 公式：

$$
\text{score}(D, Q) = \sum_{i=1}^{n} \text{IDF}(q_i) \cdot \frac{f(q_i, D) \cdot (k_1 + 1)}{f(q_i, D) + k_1 \cdot (1 - b + b \cdot \frac{|D|}{\text{avgdl}})}
$$

其中：
- $f(q_i, D)$：term $q_i$ 在文档 $D$ 中的频率
- $|D|$：文档长度，$\text{avgdl}$：平均文档长度
- $k_1$：词频饱和参数（通常 1.2-2.0）
- $b$：文档长度归一化参数（通常 0.75）
- $\text{IDF}(q_i) = \log \frac{N - n(q_i) + 0.5}{n(q_i) + 0.5}$

### 4.3 密集检索 (Dense Retrieval)

**双编码器架构**：

$$
\text{score}(q, d) = \text{cosine}(E_q(q), E_d(d))
$$

常用模型：BERT-base、DPR、BGE、M3E

### 4.4 HNSW 向量索引

- 构建多层图，每层是不同精度的近似
- 搜索时从顶层开始，逐层精化
- 时间复杂度：$O(\log N)$

## 5. 混合检索与 Rerank

### 5.1 混合检索 (Hybrid Search)

结合稀疏检索和密集检索的优势：

$$
\text{score}_{\text{hybrid}} = \alpha \cdot \text{score}_{\text{BM25}} + (1-\alpha) \cdot \text{score}_{\text{vector}}
$$

### 5.2 RRF (Reciprocal Rank Fusion)

**公式**：

$$
\text{RRF}(d) = \sum_{i=1}^{K} \frac{1}{k + \text{rank}_i(d)}
$$

其中 $k$ 是平滑因子（通常 $k=60$）。

### 5.3 Rerank 流程

```
初始检索 (Top-100) → Cross-Encoder 重排 → 最终 Top-10
```

**Cross-Encoder 优势**：同时编码 Query 和 Document，交互建模更精确。

## 6. RAG 优化策略

### 6.1 常见问题与解决方案

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 检索不到相关内容 | 块太大/嵌入不准确 | 调整 chunk size，使用更优嵌入 |
| 上下文过长 | Top-K 过多 | 上下文压缩、Self-RAG |
| 格式错误 | Prompt 不清晰 | 约束输出格式 |
| 回答不全面 | 召回不足 | 增加检索多样性 |
| 幻觉 | LLM 忽视检索内容 | 强制引用、Chain-of-Verification |

### 6.2 Query 改写技术

**HyDE (Hypothetical Document Embeddings)**：
1. LLM 生成假设性回答
2. 用回答的 embedding 检索
3. 返回真实相关文档

### 6.3 Self-RAG

Self-RAG 通过**反思 token** 自评估检索和生成质量：

| Token | 作用 |
|-------|------|
| `[检索]` | 是否需要检索 |
| `[相关]` | 检索内容是否相关 |
| `[完整]` | 回答是否完整 |
| `[支持]` | 回答是否被检索内容支持 |

## 7. RAG vs SFT

### 7.1 核心区别

| 维度 | RAG | SFT |
|------|-----|-----|
| 知识更新 | 动态更新知识库 | 需要重新训练 |
| 推理成本 | 检索 + 生成 | 直接推理 |
| 适用场景 | 需要最新知识 | 固定模式/风格 |
| 幻觉风险 | 可控（基于检索） | 依赖模型 |

### 7.2 何时用 RAG vs SFT？

**选择 RAG**：
- 知识需要频繁更新
- 需要可溯源的答案
- 长尾知识覆盖
- 垂直领域知识库

**选择 SFT**：
- 固定的输出格式/风格
- 领域推理模式复杂
- 需要模型"学会"某种能力

## 8. 高频面试题

### Q1: RAG 的工作原理是什么？

**参考答案**：

RAG (Retrieval-Augmented Generation) 由检索和生成两部分组成：

1. **检索阶段**：用户 Query 向量化，在向量数据库中检索 Top-K 相关文档
2. **生成阶段**：将检索到的文档与 Query 组合成 Prompt，LLM 基于上下文生成答案

---

### Q2: RAG 的核心组件有哪些？

**参考答案**：

| 组件 | 功能 | 关键技术 |
|------|------|----------|
| 文档加载 | 读取各种格式 | PyMuPDF, Unstructured |
| 文本分块 | 切分文档 | 固定大小/语义切分 |
| 向量化 | 文本→向量 | BGE, M3E |
| 向量存储 | 索引管理 | FAISS, Milvus |
| 检索器 | 找到相关文档 | BM25, ANN |
| 生成器 | LLM 生成答案 | GPT-4, LLaMA |

---

### Q3: 分块策略有哪些？ 如何选择？

**参考答案**：

| 策略 | 适用场景 |
|------|----------|
| 固定大小 | 通用，简单高效 |
| 语义切分 | 保持语义完整 |
| 递归切分 | 多级粒度 |
| 文档结构 | 符合写作逻辑 |

**选择建议**：问答系统 256-512 tokens，摘要 2,000-4,000 tokens。

---

### Q4: 什么是混合检索？

**参考答案**：

混合检索结合**稀疏检索（BM25）**和**密集检索（向量相似度）**：

$$
\text{score} = \alpha \cdot \text{BM25}(Q, D) + (1-\alpha) \cdot \text{cosine}(E_Q(Q), E_D(D))
$$

稀疏检索擅长精确关键词匹配，密集检索擅长语义理解，二者互补。

---

### Q5: RAG 和 Agent 的关系？

**参考答案**：

**RAG 是 Agent 的工具之一**。Agent 可以使用 RAG 获取知识，但 Agent 还具备规划、工具调用、反思等更全面的能力。

---

### Q6: 如何解决 RAG 的检索召回率低问题？

**参考答案**：

| 方法 | 描述 |
|------|------|
| Query 改写 | HyDE、Query Expansion |
| 混合检索 | BM25 + Vector |
| 增加索引 | 多种 Embedding |
| 元数据过滤 | 提前过滤不相关 |
| 知识图谱 | GraphRAG 增强 |
